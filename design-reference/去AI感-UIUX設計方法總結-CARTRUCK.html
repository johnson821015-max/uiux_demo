「去 AI 感」UI/UX 設計方法總結

CARTRUCK 營運中心改版實戰整理（2026-07）。現成範例（工作台儀表設計語言）：
claude.ai/code/artifact/b1841183…

用途：把這次商城營運中心改版用來「擺脫 AI 生成感」的做法，整理成可重複套用的原則 + 審查檢查清單，供 UI/UX 負責人納入日常設計與審查。

一、先認得「AI 感」長什麼樣

AI 生成的介面現在很容易撞同一批長相。看到下面這些，就要警覺自己是不是掉進預設：

  • 紫藍漸層 hero；warm cream（#F4F1EA）+ 襯線大標 + 磚紅點綴
  • 近黑底 + 單一螢光綠／朱紅點綴；報紙式細線多欄
  • Inter / Space Grotesk 當「安全字體」、字級沒有節奏
  • emoji 當區塊標記／icon（最明顯的一個 tell）
  • 全部置中對齊、rounded-lg 到處都是、卡片一律「左側彩條 + 柔和浮空陰影」
  • glassmorphism（毛玻璃）濫用
核心觀念：這些不是「錯」，是「預設」。 沒有明確方向時，不要把設計力氣花在這批人人都撞的長相上。有觀點的選擇，才會脫離 AI 感。

二、對治方法（九條，可重複套用）

  1. 先釘住主題（subject-grounded）。設計語言要從產品自身的世界長出來，而不是套通用模板。本案是卡車／汽車電系零件的營運後台，使用者整天在派工、出貨、對帳——所以定位成「工作台儀表／派工控制台」，不是 SaaS 落地頁。先問：這產品的材料、場域、行話是什麼？
  2. 殺掉「emoji 當 icon」。這是最強的 AI tell。狀態改用形狀編碼：待辦用嚴重度左側色條（行動＝重點色／注意＝琥珀／急件＝紅）、狀態用小圓點 chip、分類用文字標籤。只保留功能性符號（關閉／編輯／列印／勾選）與資料型圖示。
  3. Token 化 ＋ 只選「一個」有觀點的重點色。全站色彩走 CSS 變數。重點色只選一個、而且要有觀點——本案用氧化橘（#c0531a）呼應卡車／派工調性，不是隨手的品牌藍或 AI 紫。語意色（綠／琥珀／紅）與重點色分開。整站換色只動一個變數。
  4. 中性色要「選過」，不是預設灰。不要 Tailwind 冷灰一路到底。用暖石墨（中性色帶一點點主色偏向）——純中灰讀起來像「沒選過」，帶微偏的灰讀起來像「刻意挑的」。
  5. 字體要有觀點；把「數字」當一等公民。中文用系統字（塞 CJK 網路字型不切實際）。個性從數字全用等寬 + tabular-nums 來——金額／單號／SKU／庫存全部對齊＝帳冊／儀表般的專業感。對數據密集的工具尤其有效。標籤加一點字距。
  6. 髮絲線取代浮空陰影；圓角收斂。平面卡片 + 1px 髮絲線，比到處柔和陰影更俐落、更「工具」。圓角收斂到小值（近直角），拒絕 rounded-lg 無差別套用。
  7. 分頁用「標牌式底線」，不要膠囊 pill。作用態用底線標牌（active 底部一條重點色），而不是圓角膠囊背景。pill 是 AI 常見長相；標牌線更像儀器面板。
  8. 密度＝工具感。營運工具是「用來操作」的，不是「用來閱讀」的落地頁。要資訊密、摘要優先，不要大留白。落地頁的空曠感套到後台，就會顯得空又假。
  9. 平面，不要漸層。功能性 UI 上的漸層（外殼、按鈕、圖表長條）幾乎都是 AI tell。一律單色；需要層次用邊框、底色深淺、間距處理。
補充（資訊設計）：UI 是被「掃描與操作」的，不是從上讀到下。摘要先於明細；狀態用 chip／severity stripe 讓「需要注意的」一眼可辨；圖表也要用心——area fill + 淡格線 + 強調端點，別給預設長條。

三、Before → After（本案例對照）

原本（AI 味來源）改版後（去 AI 感）
每個區塊／待辦用 emoji 當 icon移除；改分類標籤 + 左側嚴重度色條，狀態用「形狀」
深灰漸層外殼 + 圓角膠囊導覽平面暖墨色控制台 + 標牌式底線分頁；圖表漸層也拿掉
一路 Tailwind 冷灰、沒有重點色暖石墨中性色 + 一個氧化橘重點色；語意色另分
到處 rounded-lg + 浮空陰影近直角 + 髮絲線分隔，密度拉高像真正在用的工具
沒有字體個性所有數字等寬對齊（帳冊感）＝個性來源

四、設計審查快速檢查清單

避免 · AVOID

  • emoji 當標題前綴／功能 icon
  • 紫藍（或任何）漸層當裝飾；毛玻璃
  • 一份設計裡兩三種以上重點色互搶
  • rounded-lg／大圓角無差別套用；每卡都浮空陰影
  • 全置中；字級沒階層；Inter 當唯一字體
  • 落地頁式大留白套到密集工具

採用 · ADOPT

  • 一個「有觀點且扣主題」的重點色，其餘克制
  • 中性色選過（帶主色微偏）
  • 狀態用形狀編碼（chip／severity stripe／dot）
  • 數字等寬 + tabular-nums；標籤加字距
  • 髮絲線 > 陰影；圓角收斂；分頁用底線標牌
  • 摘要先於明細；圖表 area fill + 淡格線 + 強調端點

五、本專案落地參考

  • 設計系統 = CSS 變數 --ct-*，定義在 wp-content/mu-plugins/cartruck-ops.php 的 admin_head <style>（:root{ --ct-paper … }）。
  • 唯一重點色 = --ct-accent:#c0531a，改這一個變數全營運中心跟著換色。淺色 only（活在淺色 wp-admin 內）。
  • 外殼（頂端控制台）單一來源在 cartruck-ops.php；別在別檔重複拷貝 CSS。
  • 詳細規則見專案 CLAUDE.md §三「營運中心設計系統」。
