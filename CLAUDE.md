# CLAUDE.md

## 這個專案是什麼

單一檔案的 UI/UX 風格基準檔（Design System Template）。`index.html` 就是全部——沒有建置流程、沒有相依套件、沒有測試。用瀏覽器直接打開即可預覽。

- Repo：`johnson821015-max/uiux_demo`
- 部署：GitHub Pages（`.github/` 內的 workflow，push 到 `main` 自動發布）

它的用途是被「當作附件丟給 AI」，讓 AI 照著它的 tokens 與元件樣式去改寫其他專案的 UI。所以這個檔案的可讀性與註解結構，跟它的視覺呈現一樣重要。

## 架構

`index.html` 約 3700 行，順序固定為 `<style>` → `<body>` 頂端列 → 展示區塊 → `<script>`。

`<style>` 內用編號註解分段（`① DESIGN TOKENS`、`② BASE`、`③ TOKEN 展示`…），編號目前排到 ⑯。注意編號**與檔案順序不完全一致**（⑮ 第三方整合、⑯ 電商版面 排在 ⑫ 版面模式 之前），這是歷次擴充留下的，改動時不要為了對齊而大搬風。

### 頂端列：三列，只承載風格

頂端列**不是元件示範**，是這份模板的操作介面，分三列：

1. **風格分頁**——切換 `<html data-style="...">`。分頁只承載風格名稱，不承載元件。
2. **數值細項面板**——調整當前風格的 token，即時寫入 `documentElement` 的行內樣式。
3. **區塊錨點導覽**——下方元件展示區的目錄。

下方所有元件跟著當前風格重繪。頂端列黏頂約 148px，`.demo-section` 因此帶 `scroll-margin-top`。

`<body>` 每個展示區塊是 `<section class="demo-section" id="...">`：

| id | 內容 |
|----|------|
| `tokens` | 色票／字級／圓角／陰影展示 |
| `buttons` `forms` `cards` | 基礎元件 |
| `commerce` | 電商版面（促銷／列表／詳情／閃購／優惠券／評價／購物車） |
| `nav` `data` `feedback` `overlay` | 導覽、資料展示、回饋、覆蓋層 |
| `motion` `interactive` | 動態效果、進階互動 |
| `mobile` | 手機 App 畫面（7 種） |
| `thirdparty` | 地圖／金流／分享／影片／客服／Cookie |
| `layout` | Hero／特色區／側欄後台／Footer |

## 核心規則：一切都走 Design Tokens

Token 分三層：

```
:root                                      → 全風格共用（字體階層、間距、動畫）
:root[data-style="x"]                      → 某風格的淺色 token
:root[data-style="x"][data-theme="dark"]   → 該風格的深色覆寫（只列差異）
```

**風格（`data-style`）與明暗（`data-theme`）是兩個正交維度。深色不是一種風格，是每種風格底下的一個模式。** 新增風格時務必補齊淺色與深色兩個區塊。

深色區塊靠選擇器特異度勝出（`(0,3,0)` > `(0,2,0)`），所以只需列出與淺色不同的 token，其餘自動沿用。

**元件樣式裡不准出現寫死的顏色、間距、圓角、陰影。** 一律引用變數：

```css
/* 對 */
.btn-primary { background: var(--primary); border-radius: var(--radius-md); padding: var(--sp-3) var(--sp-5); }

/* 錯 */
.btn-primary { background: #4F46E5; border-radius: 10px; padding: 12px 24px; }
```

例外只有一種：token 定義本身（`:root` 與 `[data-theme="dark"]` 區塊內）。

Token 分組：品牌色（`--primary` `--secondary` `--accent`）、語意色（`--success` `--warning` `--danger` `--info`，各配一個 `-bg` 淺底）、中性色（`--bg` `--surface` `--surface-2` `--border` `--text` `--text-2` `--text-3`）、字體（`--font-family` `--fs-xs`~`--fs-3xl` `--fw-*`）、間距（`--sp-1`~`--sp-8`，8px 系統）、圓角（`--radius-sm/md/lg/full`）、陰影（`--shadow-sm/md/lg`）、動畫（`--transition`）。

陰影不寫死顏色，而是由 `--shadow-rgb`（色）與 `--shadow-a`（明暗倍率）組出來，例如 `rgba(var(--shadow-rgb), calc(.06 * var(--shadow-a)))`。深色模式把 `--shadow-a` 拉到 5–7，同一組公式就能兩邊通用。

新增 token 前先確認現有的不夠用。這組變數是對外契約——AI 讀它、其他專案照它抄，隨意增刪會讓既有提示詞失效。

## 內建的三套風格

| `data-style` | 名稱 | 定位 |
|---|---|---|
| `indigo-saas` | 靛紫 SaaS | **對照組**：三色互搶、Tailwind 冷灰、大圓角、浮空陰影，正是準則逐條點名的 AI 預設 |
| `warm-graphite` | 暖石墨工作台 | **預設**：單一氧化橘、暖中性色、近直角、髮絲線。token 層已收斂，元件層仍有漸層與 emoji（混合態） |
| `instrument` | 工作台儀表 | 風格二想成為的樣子：零陰影、更近直角、髮絲線更明顯 |

## 風格引擎（`<script>` 開頭）

- 使用者的調整存成 **tweaks**，鍵是 `風格id:明暗`，以行內樣式覆寫。**淺色與深色分開記錄，互不影響**——在淺色調的主色不會自動套到深色。
- 新增風格 = 快照當前 light/dark 的生效 token，注入一份真正的 `<style>` 規則，行為與內建風格完全一致。
- 存在 `localStorage`；不可用時（某些瀏覽器的 `file://`）退回記憶體並在面板提示。
- 「匯出 CSS」吐出的區塊要**貼回 `<style>` 的 DESIGN TOKENS 區**才會永久保存。localStorage 只是暫存。

改風格引擎時記得：`themeToggle` 這個 id 被手機設定頁的開關（`phoneDarkSwitch`）連動呼叫，不要改名。

## 慣例

- **語意化 class**：`.btn-primary`、`.badge-success`、`.card-stat`。AI 對照時靠 class 名稱認元件，不要用 `.b1`、`.x-wrap` 這種。
- **深色模式**：任何新元件都必須在 `[data-theme="dark"]` 下檢查一次。只要全程用 `--surface` / `--text` / `--border`，通常自動就對。
- **JS**：原生 vanilla，無框架。互動邏輯集中在檔尾 `<script>`，用 `querySelectorAll` + 事件委派，不引入任何 CDN。
- **中文優先**：註解、展示文案、README 都用繁體中文。字型堆疊已含 `Noto Sans TC` / `PingFang TC` / `Microsoft JhengHei`。
- **自我完備**：不得引用外部 CSS/JS/字型/圖片。圖片用 CSS 漸層或 inline SVG 代替。整個檔案要能離線打開。

## 改動時

新增元件 → 找到對應的編號 `<style>` 分段補樣式，再到對應 `<section>` 補展示標記，兩邊都要有。

驗證方式是**打開瀏覽器看**，切換右上角深色模式再看一次。沒有測試可跑，不要假裝跑過。

commit 訊息用繁體中文，動詞開頭，例如「新增列表風格：樹狀列/待辦清單/分組列表」。

## 設計準則：去 AI 感

出處 `design-reference/去AI感-UIUX設計方法總結-CARTRUCK.md`（CARTRUCK 營運中心改版實戰整理，2026-07）。同目錄的 `.html` 是照這套方法排版的活範例，可直接開來看效果。

**核心觀念：紫藍漸層、毛玻璃、emoji 當 icon、到處大圓角——這些不是「錯」，是「預設」。** 沒有觀點的選擇才是 AI 感的來源。

### 九條可套用的對治方法

1. **先釘住主題**——設計語彙從產品自身的世界長出來（工具？儀表？帳冊？），不是套通用 SaaS 落地頁模板。
2. **殺掉 emoji 當 icon**——最強的 AI tell。狀態改用形狀編碼：嚴重度左側色條、小圓點 chip、文字標籤。只留功能性符號（✕ 關閉、✓ 勾選、編輯、列印）與資料型圖示。
3. **只選一個有觀點的重點色**——語意色（綠／琥珀／紅）與重點色分開，各司其職。整站換色只動一個變數。
4. **中性色要「選過」**——不要 Tailwind 冷灰一路到底。中性色帶一點主色偏向；純中灰讀起來像「沒選過」。
5. **數字當一等公民**——金額／單號／SKU／庫存全部等寬 + `tabular-nums`。中文用系統字，個性從數字對齊來。標籤（label）加字距。
6. **髮絲線取代浮空陰影；圓角收斂**——1px 分隔線比柔和陰影更俐落、更像工具。圓角收到小值（近直角）。
7. **分頁用標牌式底線**——active 底部一條重點色，不要圓角膠囊 pill。
8. **密度＝工具感**——工具是拿來操作的，不是拿來讀的。資訊密、摘要先於明細，不要落地頁式大留白。
9. **平面，不要漸層**——功能性 UI 上的漸層幾乎都是 AI tell。需要層次用邊框、底色深淺、間距。

補充：UI 是被掃描與操作的，不是從上讀到下。圖表用 area fill + 淡格線 + 強調端點，別給預設長條。

### 審查檢查清單

**避免**：emoji 當標題前綴／功能 icon｜任何裝飾性漸層、毛玻璃｜兩種以上重點色互搶｜大圓角無差別套用、每卡都浮空陰影｜全置中、字級沒階層、Inter 當唯一字體｜落地頁式大留白套到密集工具

**採用**：一個扣主題的重點色，其餘克制｜中性色帶主色微偏｜狀態用形狀編碼（chip／severity stripe／dot）｜數字等寬 + `tabular-nums`、標籤加字距｜髮絲線 > 陰影、圓角收斂、分頁用底線標牌｜摘要先於明細

### 例外：第三方品牌色

元件裡少數寫死的 hex 是第三方品牌識別色，**不受「不准寫死顏色」規則約束**，也不該 token 化：LINE `#06C755`、YouTube `#FF0000`、Google 按鈕邊框 `#DADCE0`、Instagram 漸層 `#F58529`→`#DD2A7B`。商品色票（如「石墨黑 `#1E293B`」）屬於資料，同理不動。

### 重點色的選法

`#B3500F` 是從 CARTRUCK（卡車／派工控制台）的主題長出來的——**它是範例，不是規定**。準則第一條是「先釘住主題」：套用到別的產品時，重新問這個產品的材料、場域、行話是什麼，從那裡取一個有觀點的重點色。照抄氧化橘只是換了一種預設。
